# Prompt Engineering 设计文档

## 一、项目介绍

本项目通过结构化Prompt设计，将大语言模型应用于真实业务场景。

通过角色设定、任务拆解、约束条件和输出格式设计，提高模型输出稳定性和可用性。

---

# 二、Prompt设计方法

## 1. 角色设定（Role）

为模型设置专业身份，使输出更加符合业务需求。

例如：

你是一名拥有5年经验的AI产品经理，熟悉用户需求分析、产品设计和功能规划。

---

## 2. 任务拆解（Task）

将复杂任务拆分为多个步骤。

例如：

用户反馈分析：

1. 提取用户需求；
2. 分析用户痛点；
3. 判断需求优先级；
4. 提供产品优化方案。

---

## 3. 输出约束（Constraint）

规定模型输出格式，提高结果可读性。

例如：

请按照以下结构输出：

1. 用户画像分析
2. 使用场景分析
3. 用户痛点
4. 产品优化建议

---

# 三、典型Prompt案例

## 案例1：AI产品需求分析助手

应用场景：

帮助产品经理分析用户反馈。

输入：

产品信息 + 用户反馈

输出：

- 用户画像
- 用户痛点
- 需求优先级
- 功能优化建议


---

## 案例2：AI岗位分析助手

应用场景：

分析岗位JD，辅助求职准备。

输入：

岗位描述 + 个人经历

输出：

- 岗位能力要求
- 匹配程度分析
- 简历优化建议


---

## 案例3：AI内容策划助手

应用场景：

辅助新媒体内容生产。

输入：

内容主题

输出：

- 选题方向
- 标题方案
- 内容结构
- 优化建议


---

# 四、Prompt优化过程

第一次测试：

模型输出较宽泛，缺少针对性。

优化方式：

- 增加角色设定；
- 明确任务目标；
- 增加输出格式限制；
- 增加业务背景。


优化结果：

模型输出更加稳定，更符合实际应用需求。
